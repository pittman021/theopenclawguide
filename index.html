// WHAT IS OPENCLAW?

OpenClaw is an open-source AI assistant framework that runs locally on your machine. Think of it as having a digital chief of staff that:

  • Lives in your messaging apps (Telegram, Slack, WhatsApp)
  • Can access your files, calendar, and data
  • Executes tasks on your behalf
  • Remembers context across conversations

It's not a chatbot. It's an autonomous agent that works when you don't.

What OpenClaw Is NOT

  • Not a SaaS subscription you rent
  • Not a black box you can't inspect
  • Not just for developers

What OpenClaw IS

  • Open source (you own it)
  • Runs locally (your data stays yours)
  • Customizable to YOUR workflows
  • Free to run (just pay for API tokens)

// MODULE 1: FOUNDATION

Goal: Get OpenClaw running + your first working agent

Step 1: Installation & Setup

OpenClaw runs on Mac, Linux, or Raspberry Pi. Here's the quickstart:

# Clone the repo
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# Install dependencies
npm install

# Run the setup
npm run setup

Step 2: Understanding Agents vs Sub-Agents

Agents = Independent workers with their own personality and memory
Sub-Agents = Task-specific helpers spawned by your main agent

Think of it like:

  • Agent = Your employee
  • Sub-Agent = A contractor you hire for one specific job

Step 3: Your First Agent — Daily Briefing

This is the simplest starting point. Your agent sends you a daily summary of what matters.

The prompt (copy and paste this):

You are my Personal AI Assistant. Every morning at 8am, review my calendar, tasks, and any important updates. Then provide a brief, actionable summary covering:

  1. Top 3 priorities for today
  2. Any meetings that need preparation
  3. Follow-ups or replies I might be missing

Keep it short, clear, and actionable. Format as a bullet list.

That's it. Paste that into your agent's configuration, and every morning you'll wake up to a personalized briefing. No coding required.

Step 4: Workspace Organization

// UNDERSTANDING MODELS

Goal: Choose the right model for the right task

One of OpenClaw's superpowers is that you can switch between different AI models depending on what you need. Here's how to think about it:

Model Providers

Anthropic (Claude)

  • Best for: Nuanced reasoning, writing, analysis
  • Models: Haiku (fast), Sonnet (balanced), Opus (best quality)
  • Strengths: Excellent follow-through, less likely to hallucinate

OpenAI (GPT)

  • Best for: Code, general tasks, speed
  • Models: 4o Mini (cheap), 4o (balanced), o1 (reasoning)
  • Strengths: Massive knowledge, great at following complex instructions

Minimax

  • Best for: Fast responses, cost-sensitive tasks
  • Models: MiniMax-M2.5 series
  • Strengths: Speed, competitive pricing

When to Switch Models

Task Recommended Model
Quick reminders, simple lookups Mini/Haiku
Writing, editing, brainstorming Sonnet/GPT-4o
Complex analysis, multi-step reasoning Opus/o1

Cost Management Tips

  • Default to cheaper models for routine tasks
  • Upgrade only when needed for complex requests
  • Set usage alerts in your configuration
  • Review token usage weekly

// SOUL FILE STRUCTURE

Goal: Understand and customize your agent's identity

This is where most people get stuck. The docs don't explain it well, so here's the deep dive.

What is SOUL.md?

SOUL.md defines WHO your agent is. Not what it does — WHO it is. Think of it as the agent's personality, values, and voice.

What is TOOLS.md?

TOOLS.md is your LOCAL setup — camera names, SSH aliases, TTS preferences. Things specific to YOUR environment that shouldn't be shared.

What is USER.md?

USER.md defines who YOU are — your preferences, context, goals. This helps your agent tailor its responses to you.

The File Structure

workspace/
├── SOUL.md          ← Agent identity (personality, voice)
├── USER.md          ← Who you're helping (preferences, context)
├── TOOLS.md         ← Your local setup (environment-specific)
├── MEMORY.md        ← Long-term memory (consolidated)
├── BRAIN.md         ← Active projects and action items
├── memory/          ← Daily logs
│   └── YYYY-MM-DD.md
├── agents/          ← Sub-agents with their own configs
│   ├── warren/
│   │   ├── SOUL.md
│   │   └── PORTFOLIO.md
│   └── crawly/
│       ├── SOUL.md
│       └── SKILL.md
└── skills/          ← Shared skills

Key Principle

SOUL = WHO (identity, personality)

SKILL = HOW (workflow, tools, outputs)

TOOLS = WHERE (your local environment)

How to View and Edit Files

OpenClaw makes it easy to see and modify your agent's files:

  1. View files: Ask your agent "show me SOUL.md" or use the file browser
  2. Edit files: Say "update SOUL.md to include..." or edit directly
  3. Create new agents: Add folders in agents/ with their own SOUL.md

// MEMORY & CONTEXT

Goal: Ensure your agent remembers across sessions

This is critical. Without proper memory setup, your agent forgets everything between sessions.

How Memory Works

OpenClaw has three memory layers:

  1. Short-term: Conversation context (resets each session)
  2. Medium-term: Daily memory files in memory/YYYY-MM-DD.md
  3. Long-term: MEMORY.md for consolidated persistent knowledge

Setting Up Memory

Memory should be ON by default. To verify:

  1. Check that memory/ folder exists in your workspace
  2. Confirm MEMORY.md is present
  3. Test it: ask your agent something, start a new session, ask again

Best Practice: Always Update Memory

Before ending any session, tell your agent to:

"Update memory with what we worked on today and what's pending"

Or manually add to memory/YYYY-MM-DD.md:

  • What you worked on
  • Decisions made
  • What's next
  • Any important context

Starting a New Session

When you start a new session, OpenClaw automatically reads:

  • SOUL.md
  • USER.md
  • MEMORY.md
  • Today's and yesterday's memory files

This is why memory files are essential — they preserve context across sessions.

// SKILLS

Goal: Learn how to use and create skills

Skills are reusable tool sets that give your agent new capabilities.

What Are Skills?

A skill is a collection of instructions and tools bundled together. Think of it like an app for your agent.

Built-in Skills

OpenClaw comes with useful skills:

  • Weather — Get forecasts
  • Healthcheck — Security auditing
  • Things — Task management integration

How to Enable a Skill

  1. Check the skills folder (skills/)
  2. Read the SKILL.md for the skill you want
  3. Enable it in your configuration
  4. Test with a simple command

Creating Custom Skills

Create a SKILL.md file in your skill folder:

# My Custom Skill

## What This Does
Brief description of the skill's purpose

## How to Use
- Command 1: What it does
- Command 2: What it does

## Tools Used
- tool1
- tool2

Skill Best Practices

  • Keep skills focused — One skill = one capability
  • Document clearly — Include usage examples
  • Test thoroughly — Make sure it works before relying on it
  • Share with community — Others might find it useful!

// USE CASE: EMAIL CLEANUP

Goal: Archive and delete thousands of emails in bulk

Got thousands of emails piling up? An agent can help you organize, archive, and delete in bulk — way faster than manual.

What This Agent Does

  • Scan your inbox for old/unwanted emails
  • Categorize by sender, age, subject
  • Archive to storage folder (keep for records)
  • Delete trash with your approval
  • Report what it did (emails archived, space freed)

Setup

Email Agent SOUL

# Email Cleanup Agent

## Persona
You are a meticulous email organizer.

## Rules
- Always ask before deleting
- Archive instead of delete when uncertain
- Report space savings

## Capabilities
- List emails by age, sender, label
- Archive in bulk
- Delete in bulk (with approval)

Email Provider Setup

Provider Method Notes
Gmail Gmail API Need OAuth setup
Outlook Microsoft Graph API Need OAuth setup
Apple Mail Local access Easiest on Mac

Workflow

  1. Ask: "Clean up emails older than 2 years from Newsletter senders"
  2. Agent scans: Finds 3,400 matching emails
  3. Agent shows: "Found 3,400 emails. Archive or delete?"
  4. You approve: "Archive them"
  5. Agent does it: Archives all, reports "Freed 230MB"

When to spawn a sub-agent vs. direct execution:

Task Type Approach
Quick one-off (lookup, format) Direct execution
Complex multi-step Sub-agent

// EXPANDING YOUR AGENT'S BRAIN

Goal: Connect your second brain for persistent knowledge

Your agent's memory is powerful, but it's even better when it can access your existing knowledge base.

Option 1: Obsidian (Recommended)

Obsidian is a markdown-based note-taking app that stores plain text files. Perfect for AI access.

Option 2: Notion

Notion is great for structured data. Use the API to connect.

Option 3: Apple Notes

Already using Notes? OpenClaw can read from Apple Notes via the skill.

The Hybrid Approach

  • Obsidian: Long-term knowledge, projects, meeting notes
  • OpenClaw memory/: Agent-specific daily logs
  • MEMORY.md: Consolidated context summary
  • BRAIN.md: Active projects and action items
