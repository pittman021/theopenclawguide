// HOW OPENCLAW WORKS

Before we dive in, let's look under the hood. Understanding how OpenClaw works will help you customize it to your needs.

The Basic Flow

When you message your agent, here's what happens:

  1. You send a message (via Telegram, Slack, WhatsApp, or terminal)
  2. OpenClaw reads your files — SOUL.md, USER.md, MEMORY.md
  3. It understands your request — using the AI model
  4. It takes action — calls tools, runs skills, spawns sub-agents
  5. It remembers — writes back to MEMORY.md
  6. You get a response — personalized to you

The Core Files

Four files make your agent work:

  • SOUL.md — WHO your agent is (personality, voice, identity)
  • USER.md — WHO you're helping (your preferences, context)
  • MEMORY.md — Long-term memory (what agent remembers forever)
  • memory/ — Daily logs (YYYY-MM-DD.md files)

Sub-Agents

Your main agent can spawn smaller agents for specific tasks:

  • Warren — monitors investments, tracks stocks
  • Crawly — researches topics, gathers info
  • Cathie — creates content, writes posts

Think of sub-agents as contractors you hire for one-off jobs, while your main agent is your full-time employee.

Skills

Skills are like apps that give your agent new abilities — weather, email, calendar, notes, and more.

// SECTION 1: SETUP

How to Setup Your AI Agent in 30 Minutes

Let's get OpenClaw running on your machine. We'll cover installation, dashboard navigation, and sending your first command.

Installation

OpenClaw runs on Mac, Linux, or Raspberry Pi. Here's the quickstart:

# Clone the repo
git clone https://github.com/openclaw/openclaw.git
cd openclaw

# Install dependencies
npm install

# Run the setup
npm run setup

Connect Your Messaging App

OpenClaw lives in your messaging apps. Here's how to connect:

Telegram

  1. Message @BotFather on Telegram
  2. Create a new bot with /newbot
  3. Copy your API token
  4. Add it to your OpenClaw config

Slack

  1. Go to Slack API → Create App
  2. Enable Bot Token Scope
  3. Install to your workspace
  4. Copy Bot User OAuth Token

WhatsApp

  1. OpenClaw supports WhatsApp Business API
  2. Connect via QR code scanning
  3. Follow the in-app setup wizard

Navigate the Dashboard

Once running, your dashboard shows:

  • Agents: Your active AI agents
  • Skills: Available capabilities
  • Memory: Conversation history
  • Settings: Configuration options

Send Your First Command

Open your messaging app and try:

"Hey, what's the weather today?"

Your agent is now live and responding!

Your First Agent — Daily Briefing

Let's create your first useful agent. It'll send you a daily summary every morning.

The prompt (copy and paste):

You are my Personal AI Assistant. Every morning at 8am, review my calendar, tasks, and any important updates. Then provide a brief, actionable summary covering:

  1. Top 3 priorities for today
  2. Any meetings that need preparation
  3. Follow-ups or replies I might be missing

Keep it short, clear, and actionable. Format as a bullet list.

That's it. Every morning you'll wake up to a personalized briefing. No coding required.

// SECTION 2: UNDERSTANDING MODELS

Choose the right model for the right task

One of OpenClaw's superpowers is switching between AI models depending on what you need.

Model Providers

Anthropic (Claude)

  • Best for: Nuanced reasoning, writing, analysis
  • Models: Haiku (fast), Sonnet (balanced), Opus (best quality)
  • Strengths: Excellent follow-through, less likely to hallucinate

OpenAI (GPT)

  • Best for: Code, general tasks, speed
  • Models: 4o Mini (cheap), 4o (balanced), o1 (reasoning)
  • Strengths: Massive knowledge, great at following complex instructions

Minimax

  • Best for: Fast responses, cost-sensitive tasks
  • Models: MiniMax-M2.5 series
  • Strengths: Speed, competitive pricing

When to Switch Models

Task Recommended Model
Quick reminders, simple lookups Mini/Haiku
Writing, editing, brainstorming Sonnet/GPT-4o
Complex analysis, multi-step reasoning Opus/o1

Cost Management Tips

  • Default to cheaper models for routine tasks
  • Upgrade only when needed for complex requests
  • Set usage alerts in your configuration
  • Review token usage weekly

// SECTION 3: BUILDING YOUR BOT'S SOUL

Understand and customize your agent's identity

This is where most people get stuck. The docs don't explain it well, so here's the deep dive.

What is SOUL.md?

SOUL.md defines WHO your agent is. Not what it does — WHO it is. Think of it as the agent's personality, values, and voice.

What is TOOLS.md?

TOOLS.md is your LOCAL setup — camera names, SSH aliases, TTS preferences. Things specific to YOUR environment that shouldn't be shared.

What is USER.md?

USER.md defines who YOU are — your preferences, context, goals. This helps your agent tailor its responses to you.

The File Structure

workspace/
├── SOUL.md          ← Agent identity (personality, voice)
├── USER.md          ← Who you're helping (preferences, context)
├── TOOLS.md         ← Your local setup (environment-specific)
├── MEMORY.md        ← Long-term memory (consolidated)
├── BRAIN.md         ← Active projects and action items
├── memory/          ← Daily logs
│   └── YYYY-MM-DD.md
├── agents/          ← Sub-agents with their own configs
│   ├── warren/
│   │   ├── SOUL.md
│   │   └── PORTFOLIO.md
│   └── crawly/
│       ├── SOUL.md
│       └── SKILL.md
└── skills/          ← Shared skills

Key Principle

SOUL = WHO (identity, personality)

SKILL = HOW (workflow, tools, outputs)

TOOLS = WHERE (your local environment)

How to View and Edit Files

OpenClaw makes it easy to see and modify your agent's files:

  1. View files: Ask your agent "show me SOUL.md" or use the file browser
  2. Edit files: Say "update SOUL.md to include..." or edit directly
  3. Create new agents: Add folders in agents/ with their own SOUL.md

// SECTION 4: MEMORY & CONTEXT

Ensure your agent remembers across sessions

This is critical. Without proper memory setup, your agent forgets everything between sessions.

How Memory Works

OpenClaw has three memory layers:

  1. Short-term: Conversation context (resets each session)
  2. Medium-term: Daily memory files in memory/YYYY-MM-DD.md
  3. Long-term: MEMORY.md for consolidated persistent knowledge

Setting Up Memory

Memory should be ON by default. To verify:

  1. Check that memory/ folder exists in your workspace
  2. Confirm MEMORY.md is present
  3. Test it: ask your agent something, start a new session, ask again

Best Practice: Always Update Memory

Before ending any session, tell your agent to:

"Update memory with what we worked on today and what's pending"

Or manually add to memory/YYYY-MM-DD.md:

  • What you worked on
  • Decisions made
  • What's next
  • Any important context

Starting a New Session

When you start a new session, OpenClaw automatically reads:

  • SOUL.md
  • USER.md
  • MEMORY.md
  • Today's and yesterday's memory files

This is why memory files are essential — they preserve context across sessions.

Learn how to use and create skills

What Are Skills?

A skill is a collection of instructions and tools bundled together. Think of it like an app for your agent.

Built-in Skills

OpenClaw comes with useful skills:

How to Enable a Skill

  1. Check the skills folder (skills/)
  2. Read the SKILL.md for the skill you want
  3. Enable it in your configuration
  4. Test with a simple command

Creating Custom Skills

Create a SKILL.md file in your skill folder:

# My Custom Skill

## What This Does
Brief description of the skill's purpose

## How to Use
- Command 1: What it does
- Command 2: What it does

## Tools Used
- tool1
- tool2

Skill Best Practices

  • Keep skills focused — One skill = one capability
  • Document clearly — Include usage examples
  • Test thoroughly — Make sure it works before relying on it
  • Share with community — Others might find it useful!

Email Integration (Gmail)

OpenClaw can read, summarize, draft, and send emails through your Gmail account. Here's how to set it up safely.

Why a Dedicated Email Matters

Never connect your personal Gmail to OpenClaw. Instead, create a brand new Gmail account just for your bot. This keeps your personal data safe and isolated. If anything goes wrong, it only affects the bot account — not your main email, calendar, or Google services.

Google Workspace Option

If you have a Google Workspace account, that counts as a completely separate account from your personal Gmail. You can connect it to OpenClaw independently.

For custom domains (like bot@yourdomain.com), you have two options:

  • Google Workspace — ~$6/month, gives you Gmail with your own domain
  • Resend — Free tier: 3,000 emails/month, works with OpenClaw's Resend skill

Step-by-Step Setup

  1. Create a bot Gmail account — Go to gmail.com, create a new account just for your bot. Save the credentials in a password manager.
  2. Enable Gmail API — Go to console.cloud.google.com logged in as your bot account. Create a new project, then go to APIs & Services → Library → find Gmail API and enable it.
  3. Create OAuth credentials — Under APIs & Services → Credentials, create an OAuth client ID (Desktop app). Download the JSON file. Also add your bot Gmail as a test user in the OAuth consent screen.
  4. Install and authenticate — Run clawdhub install gog to install the Google Workspace skill, then run gog auth and follow the browser login flow using your bot Gmail account.
  5. Test it — Ask OpenClaw to check your inbox and summarize new emails. If it works, you're fully set up!

Quick Recap

Create a dedicated Gmail → Set up Google Cloud project with Gmail API → Create OAuth credentials → Authenticate with gogcli → Test. The Google Cloud setup is the most involved part but you only do it once. After that, OpenClaw can handle all your email needs.
